# SNR EduOS — Student Portal · UI / Design Specification

> **Назначение файла.** Это визуальная спецификация интерфейса для Claude Code.
> Используется вместе с `tz.md`. Если возникает конфликт: **`tz.md` — источник правды по
> функционалу, ролям, БД и приоритетам**; **этот файл — источник правды по внешнему виду,
> компоновке экранов и дизайн-токенам**. Расхождения между двумя файлами перечислены
> в конце (раздел «РАСХОЖДЕНИЯ С tz.md»).

---

## 0. ОБЩЕЕ ВПЕЧАТЛЕНИЕ ОТ МАКЕТА

Современная образовательная платформа для ученика. Стиль — «дружелюбный futuristic / EdTech»:
светлые градиентные фоны, плотные карточки со скруглёнными углами, мягкие тени, обильные
цветные иконки-плитки, 3D-иллюстрации роботов и маскот-персонажи. Тон детский-подростковый,
но не инфантильный. Основной экран приложения — десктоп/планшет с боковым меню;
мобильная адаптация обязательна (mobile-first по `tz.md`).

Всего на макете **20 экранов** + отдельная панель **Control Center** (системная боковая шторка).

## 1. ДИЗАЙН-СИСТЕМА (DESIGN TOKENS)

### 1.1 Цвета

| Токен | Значение (прибл.) | Применение |
|---|---|---|
| `--primary` | насыщенный синий `#2D5BFF` … `#3B6FF5` | основные кнопки, активные пункты меню, акценты |
| `--primary-gradient` | сине-фиолетовый градиент (login-панель, hero-блоки) | левая панель логина, полноэкранный отсчёт |
| `--bg-app` | очень светлый сине-серый `#F4F6FB` / `#EEF2FB` | общий фон рабочей области |
| `--bg-card` | белый `#FFFFFF` | карточки, панели |
| `--bg-card-soft` | бело-сиреневые мягкие градиенты | карточки предметов, KPI |
| `--text-primary` | тёмно-синий/графит `#1A2138` | заголовки, основной текст |
| `--text-muted` | серо-синий `#8A93A8` | подписи, мета-данные |
| `--success` | зелёный `#2DBE7E` | статус «Активен», «Сейчас», положительные чипы |
| `--warning` | жёлто-оранжевый `#F5A623` | «Скоро», предупреждения, средние оценки |
| `--danger` | красный/коралл `#F0556B` | «Просрочено», «Должник», ошибки |
| `--info` | голубой `#39B6F5` | информационные чипы, ссылки |

Акцентные иконки предметов имеют свои цвета (Робототехника — синий, Информатика — фиолетовый,
Математика — оранжевый, Физика — циан, Английский — розовый/красный, История — коричнево-жёлтый).
Закрепить цвет-на-предмет в конфиге (см. §3 «Цвет на предмет»).

### 1.2 Типографика

- Шрифт: **Inter** (по `tz.md`), запасной — system-ui.
- Заголовки экранов: 22–26px, bold.
- Заголовки карточек: 16–18px, semibold.
- Основной текст: 14px.
- Мета/подписи: 12–13px, `--text-muted`.

### 1.3 Геометрия и эффекты

- Радиус карточек: `16px` (крупные панели — до `20–24px`).
- Радиус кнопок: `10–12px`, у иконок-плиток — `12–14px`.
- Радиус чипов: `999px` (полные пилюли).
- Тени: мягкие, рассеянные (`0 8px 24px rgba(40,60,120,0.08)`).
- Отступы карточек: внутренний padding `16–20px`, gap между карточками `16px`.

### 1.4 Чипы статусов (status chips) — единый компонент

| Текст | Цвет |
|---|---|
| `Сейчас` / `Активен` / `Подключено` | success (зелёный) |
| `Скоро` | warning (жёлтый) |
| `Прошёл` / нейтральный | серый |
| `Просрочено` / `Должник` | danger (красный) |
| `На проверке` | info (голубой) |

## 2. ОБЩИЙ КАРКАС ПРИЛОЖЕНИЯ (APP SHELL)

Все экраны после логина используют единый каркас:

- **Левый сайдбар, ~220px** (по `tz.md`), на белом/светлом фоне.
  - Сверху: логотип-иконка + блок даты/времени («ОФ-30, среда, 14 мая»).
  - Пункты меню (сверху вниз, как на макете 2):
    1. Главная
    2. Уроки
    3. Задания
    4. Материалы
    5. Книги
    6. Кружки
    7. Проекты
    8. Прогресс
    9. AI-помощник
    10. Портфолио
    11. Тесты
    12. Настройки
  - Активный пункт подсвечен синим фоном-пилюлей + белая иконка/текст.
- **Верхняя панель (topbar)** справа на каждом экране: иконки уведомлений (с красным badge-счётчиком), доп. иконка, аватар ученика.
- **Контентная область** на фоне `--bg-app`, прокручиваемая.

Мобильно: сайдбар сворачивается в нижний таб-бар или бургер-меню; topbar остаётся.

## 3. ЭКРАНЫ (детально, в порядке макета)

### Экран 1 — Вход в систему (Login)
- Двухпанельный layout. **Левая панель** — крупный сине-фиолетовый градиент с 3D-кубом-логотипом, текстом «SNR EduOS / Платформа твоего будущего», копирайт «© 2025 SNR EduOS» внизу.
- **Правая панель (форма)** на белом:
  - Заголовок «Вход для ученика».
  - Поле «Логин» (плейсхолдер вида `Adilbek_07`).
  - Поле «Пароль» (с точками-маской).
  - Чекбокс «Запомнить меня».
  - Кнопка «Войти» — широкая, синяя, primary.
  - Ссылка «Забыли пароль?».
- Auth: Supabase Auth (логин/телефон + пароль или magic link).

### Экран 2 — Главная страница ученика (Dashboard)
- Заголовок-приветствие: «Привет, Адилбек! 👋».
- **Три KPI-карточки в ряд** (высота ~100–120px):
  1. «Следующий урок» — иконка предмета, название (Робототехника), время (09:00–09:45), кабинет (305).
  2. «Мои задания» — счётчик активных (напр. «3 активных»).
  3. «Прогресс недели» — кольцевой прогресс (напр. 68%).
- **Блок «Факт дня»** — широкая карточка с иллюстрацией-маскотом и текстом интересного факта.
- **Блок «Мои предметы»** — горизонтальный ряд цветных иконок-плиток предметов (Робототехника, Математика, Информатика, Физика, Английский).
- **Блок «Недавние материалы»** — список последних файлов/материалов со ссылками.
- Низ страницы: статус-бар с очками/баллами (геймификация — напр. «12500 баллов», «5 уровень») — *(см. расхождения)*.

### Экран 3 — Уроки (список уроков)
- Заголовок «Уроки (список уроков)».
- **Сегменты-табы**: `Сегодня` / `Неделя` / `Все предметы`.
- Подзаголовок с датой: «Среда, 14 мая», стрелка перехода.
- **Левая колонка — лента уроков по времени**: строки `время | предмет (иконка+название) | кабинет | статус-чип`.
  - Примеры: 09:00 Робототехника, каб.305, чип «Сейчас» (зелёный); 10:00 Математика, каб.301, «Скоро»; и т.д.
- **Правая колонка — «Домашние задания»**: компактный список ДЗ по предметам с дедлайнами («до 16 мая»), каждая строка кликабельна (стрелка).

### Экран 4 — Страница урока (обзор)
- **Hero-карточка урока** на градиенте: название предмета («Робототехника и электроника»), номер и тема урока («Урок 12. Управление светодиодом и кнопкой»), преподаватель (имя + аватар «Иван Петрович»), кабинет, иллюстрация платы/робота.
  - Полоса «Прогресс урока» (напр. 5/7 этапов).
- **Блок «Этапы урока»** — ряд круглых иконок-шагов: Цель · Теория · Практика · Задание · Проверка · Итог.
- **Блок «Материалы урока»** — иконки-плитки файлов (Презентация, Инструкция, Схемы, Примеры кода).
- **Блок «Задание урока»** — карточка с описанием задания + кнопка «Начать задание» (primary).

### Экран 5 — Задания
- Заголовок «Мои задания».
- **Табы-статусы**: `Активные` / `На проверку` / `Выполненные` / `Просроченные`.
- **Левая колонка — список заданий**: карточка-строка `иконка предмета | предмет | название задания | дедлайн`.
  - Пример: Робототехника — «Проект "Умный дом"» — до 16 мая.
- **Правая колонка — «Статистика»**:
  - Кольцевая диаграмма с общим числом заданий (напр. «12»).
  - Легенда: Активные / На проверке / Выполненные / Просроченные с числами.
  - Карточка «Совет дня» с маскотом внизу.

### Экран 6 — Материалы
- Заголовок «Материалы» + строка поиска «Поиск материалов…».
- **Фильтр-табы**: `Все` / `Презентации` / `Видео` / `PDF` / `Книги` / `Ссылки`.
- **Сетка карточек-плиток материалов** (≈4 в ряд): каждая — крупная иконка/превью по типу, название, подпись (предмет/тип/дата).
  - Типы на макете: Лабораторная работа, Конспект, Статья, Видео, Презентация, Схемы, Инструкция, Код.
- Внизу секция «Недавно открытые» + ссылка «Показать ещё».

### Экран 7 — Книги и учебники
- Заголовок «Книги и учебники».
- **Табы**: `Мои книги` / `Библиотека школы` / `Избранное`.
- **Сетка обложек книг** (≈4 в ряд) — яркие иллюстрированные обложки с названием предмета и классом (Алгебра 7 класс, Физика 7 класс, Информатика, История, Английский язык, Геометрия, Биология, Химия).
  - Под обложкой — тип (Конспект/Сборник/Библиотека) и название.
- Кнопка «Открыть библиотеку» (primary) внизу.

### Экран 8 — Кружки (запись на кружки)
- Заголовок «Кружки (запись на кружки)».
- **Табы**: `Все кружки` / `Мои кружки` / `Заявки`.
- **Левая колонка — список кружков**: строка `иконка | название | расписание (дни/время) | свободные места | кнопка "Записаться"`.
  - Примеры: Робототехника, 3D-моделирование, Программирование, Английский клуб.
- **Правая колонка — детальная карточка кружка**: иллюстрация (робот-конструктор), описание, расписание, преподаватель, кнопка «Подробнее»/«Подать заявку» (primary).

### Экран 9 — Проекты
- Заголовок «Проекты».
- **Табы**: `Мои проекты` / `В работе` / `Завершённые`.
- **Сетка карточек проектов** (≈3 в ряд): крупная сгенерированная иллюстрация-превью, название проекта (Умный дом, Робот-манипулятор, Сайт-портфолио, Метеостанция, Игра на Python), прогресс-бар с % (75%, 80%, 40%, 90%).
- Карточка-плейсхолдер «Новый проект» с иконкой «+».

### Экран 10 — AI-помощник
- Заголовок «AI-помощник».
- Приветствие маскота-робота: «Привет, Адилбек! Чем я могу помочь тебе сегодня?».
- **Ряд быстрых действий-чипов**: «Объяснить тему», «Помочь с заданием», «Проверить код», «Подобрать материал».
- **Пузырь-ответ ассистента** с текстом + при необходимости встроенной схемой/изображением (на макете — схема платы Arduino).
- **Поле ввода внизу** «Напиши свой вопрос…» с иконками вложения/отправки.
- *(Реализация: интеграция с Claude API — см. расхождения и `tz.md` V2.)*

### Экран 11 — Мой прогресс
- Заголовок «Мой прогресс. Общий прогресс».
- **Ряд KPI-плиток**: Объекты/Уроки, Задания, Навыки, Активность — каждая с числом и % и цветной иконкой.
- **Блок «Прогресс по предметам»** — таблица: предмет | звёздный рейтинг | % (Робототехника 85%, Информатика 75%, Математика 70%, Английский 60%, История 55%). Кнопка «Смотреть всё».
- **Блок «Достижения»** — список бейджей/наград справа.

### Экран 12 — Навыки
- Заголовок «Навыки. Виды навыков».
- **Радар-диаграмма (spider chart)** в центре: оси — Логическое мышление, Программирование, Командная работа, Работа в команде, Решение проблем и т.д., с % по каждой.
- **Правая колонка «Рекомендации»** — карточки-советы: «Прокачай навык …», «Реши больше задач на логику …» с иконками и описанием.
- Кнопка «Все навыки».

### Экран 13 — Тесты и квизы
- Заголовок «Тесты и квизы».
- **Табы**: `Активные тесты` / `Пройденные`.
- **Список тестов** — строка: иконка | название теста (Тест по алгебре, Физика. Электричество, Информатика. Python, Робототехника. Датчики) | дедлайн | прогресс % или кнопка «Продолжить» / результат.
- Кнопка «Все тесты» внизу.

### Экран 14 — Расписание
- Заголовок «Расписание. Расписание на неделю».
- **Недельный переключатель дней** (Пн12 / Вт13 / Ср14(активен) / Чт15 / Пт16) + мини-календарь месяца справа (Май 2025) с подсвеченной датой.
- **Левая колонка — уроки выбранного дня** по времени (08:00 Робототехника каб.305 «Сейчас», 10:00 Математика, 11:00 Информатика, 12:00 Физика, 13:00 Английский).
- **Правая колонка — «Ближайшие события»**: контрольная, сдача проекта, школьная олимпиада — с временем/датой и иконками.

### Экран 15 — Портфолио
- Заголовок «Портфолио».
- **Табы**: `Проекты` / `Достижения` / `Сертификаты`.
- **Левая часть — сетка карточек работ/достижений** с иллюстрациями (Умный дом, Робот-манипулятор, Сайт-портфолио) и ряд бейджей-наград.
- **Правая часть — карточка сертификата**: рамка, «СЕРТИФИКАТ», текст «За участие в олимпиаде по робототехнике», имя ученика (Адилбек Р.), дата/печать.
- Кнопки «Все достижения» / «Все сертификаты».

### Экран 16 — Уведомления
- Заголовок «Уведомления».
- **Фильтр-табы**: `Все` / `Задания` / `Уроки` / `Система`.
- **Список уведомлений** — строка: иконка типа | текст уведомления | время («5 мин назад», «сейчас», «2 часа назад», «вчера»).
  - Примеры: «Новое задание по информатике», «Начинается урок Робототехника», «Фидбек по проекту "Умный дом"», «Напоминание: тест по алгебре», «Новость школы».
- Кнопка «Отметить все прочитанными» внизу.

### Экран 17 — Настройки
- Заголовок «Настройки».
- **Под-табы**: `Профиль` / `Безопасность` / `Уведомления` / `Интерфейс`.
- **Левая часть — карточка профиля**: аватар, имя (Адилбек Рахимов), класс (7 класс), email.
- **Правая часть — настройки интерфейса**:
  - «Тема интерфейса» — выбор из цветных кружков-тем.
  - «Язык» — селект (Русский) → **заложить i18n: ru / uz / en**.
  - «Часовой пояс» — селект (UTC+5.00).
  - Тумблер «Показывать подсказки».
  - Кнопка «Сохранить изменения» (primary), «Редактировать профиль».

### Экран 18 — Помощь и поддержка
- Заголовок «Помощь и поддержка».
- **Левая колонка «Частые вопросы»** — список вопросов-аккордеонов: «Как сдать задание?», «Как подключиться к уроку?», «Где найти материалы?», «Как записаться на кружки?».
- **Правая колонка «Связаться с поддержкой»** — карточки-действия: «Написать в чат», «Отправить письмо», «Позвонить в школу» с иконками.
- Внизу — «Гид по платформе / Смотреть видеоинструкции».

### Экран 19 — Полноэкранный отсчёт (начало урока)
- Полноэкранный сине-фиолетовый градиент-оверлей.
- Текст «Сейчас начинается урок», название («Робототехника и электроника»), тема урока, преподаватель + аватар, кабинет.
- **Крупный круг-таймер обратного отсчёта** (цифра «5»), 3D-маскот-робот сбоку.
- Подпись «Переход в режим урока через 5 секунд» + кнопка «Перейти сейчас».

### Экран 20 — Режим урока (рабочая среда)
- Полноэкранный рабочий режим во время урока.
- **Шапка**: название урока, тема, таймер урока (08:24:12), кнопки «Завершить» / «Завершить урок».
- **Левая колонка — «Этапы урока»** (вертикальный степпер): 1 Цель урока, 2 Теория, 3 Практика, 4 Задание(активен), 5 Проверка, 6 Итог + «Изменения».
- **Центр — рабочая зона задания**: описание задания («Соберите схему управления светодиодом и кнопкой…») + крупная схема/изображение (макетная плата Arduino).
- **Правая колонка — «Материалы»**: плитки (Презентация, Схема, Инструкция, Примеры кода).
- **Низ — панель навигации**: «Назад» / «Вперёд» / «Сдать работу» (primary) / «AI-помощник».

## 4. CONTROL CENTER (системная шторка)

Отдельная узкая правая панель (как iOS Control Center), оверлеем поверх контента.
- Сверху: крупные часы «08:30», дата «Среда, 14 мая 2025».

## 7. ТЕХНИЧЕСКИЕ ОГРАНИЧЕНИЯ (из `tz.md`, продублировано для контекста UI)

- Стек: Next.js (App Router) + Tailwind + Supabase (Postgres/Auth/Storage/Realtime) + PWA.
- Mobile-first, адаптивность обязательна; десктоп — вторичен по приоритету загрузки, но макет рисован под десктоп — поддержать оба.
- i18n заложить сразу (ru приоритет, uz/en позже).
- RLS: ученик видит только свои данные — UI не должен предполагать доступ к чужим записям.
- Без чувствительных данных в localStorage.
- PWA: manifest + service worker (offline-кэш расписания и материалов) + Web Push (Nice-to-have).
