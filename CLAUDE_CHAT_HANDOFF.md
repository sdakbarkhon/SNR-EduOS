## 1. О ПРОЕКТЕ

**SNR EduOS — образовательная платформа частной школы/учебного центра** (Ташкент, UTC+5). Основной язык интерфейса — русский, i18n на ru/uz/en заполняется сразу для новых строк во всех трёх файлах.

**Стек и структура (монорепо pnpm + Turborepo):**
- `apps/web` — Student Portal, Next.js 15 (App Router) + Tailwind + Supabase. Роли: ученик, учитель, родитель (частично), админ/суперадмин. Деплой на Vercel. **Стабилен, всё в проде.**
- `apps/mobile-parent` — приложение для **родителей** (не учеников), Expo/React Native. iOS — через Expo Go (OTA `eas update` на канал `preview`, реальной `.ipa` пока нет — нужен Apple Developer аккаунт, ~$99/год, заказчик пока не оформил). Android — полноценный APK через `eas build`. **← фокус следующей сессии.**
- `apps/mobile` — ученическое приложение. **ЗАМОРОЖЕНО НАВСЕГДА**, не развивать без явного запроса.
- `apps/h5p` — не в скоупе, не трогать без явного запроса.
- `packages/core` — общий TS-слой: типы, Supabase-клиент, query-функции, i18n-словари (`src/i18n/{ru,en,uz,types}.ts`), конфиг предметов (`src/config/subjects.ts`).
- `supabase/migrations` — SQL-миграции, **последняя применённая — 146**, следующий свободный номер — **147**. Миграции создаются как файлы в репозитории, но применяются к hosted-БД **вручную заказчиком** через Supabase Dashboard SQL Editor (Claude Code не имеет и не должен иметь прямого доступа на запись к hosted Postgres в обычном режиме работы).

**Изоляция данных** — через RLS-политики (Postgres Row Level Security), не через клиентскую логику.

### Что готово и стабильно работает

1. **Ручное управление уроками** — и учитель, и ученик могут начать/закончить урок отдельными кнопками (одни и те же `startLesson()`/`endLesson()`). «Выйти из урока» у ученика — отдельное, НЕ завершающее урок действие (чистая навигация). Статус «Пропущен» убран целиком (его не было в БД, только в UI); прошедший-непроведённый урок показывается как «Запланирован». Метка «Сейчас» замирает на последнем уроке дня до полуночи. Крон `close-past-lessons` (19:00 UTC = 00:00 Ташкент) закрывает только вчерашние-и-старше уроки.
2. **30 учеников** (по 10 на группу 10-А/7-А/3-А), сокращено с 96 — критерий: полнота данных, не происхождение username. Демо-вход всегда попадает на один из этих 30 полных аккаунтов.
3. **Презентация у ученика без блокирующего оверлея** — full-width слайд с локальным скроллом, отдельный «режим фокуса» (не браузерный Fullscreen API). Realtime-синхронизация слайдов (`SlideViewer.tsx`, канал `stage-slide-*`) не тронута.
4. **AI-ассистент ученика**: LaTeX-формулы (KaTeX) и графики функций (`function-plot` + безопасный `mathjs`-парсер, не `eval`) прямо в чате.
5. **Чат — Telegram-раскладка**: три независимые зоны скролла, умный автоскролл. Правка была не в самом чате, а в трёх ролевых шеллах (`AppShell`/`TeacherShell`/`ParentShell`), у всех был безусловный `overflow-y-auto` на `<main>`.
6. **Учебные планы**: карточка кликабельна → страница деталей с правкой тем + 3 способа создания уроков из темы (авто с 1 августа / вручную по одному / заготовка в форму). Идемпотентность — через `lessons.curriculum_topic_id`.
7. **Раздел «Задания» — полностью пересобран** (см. решения 10-13 в `resheniya_2.md`):
   - Все задания пересозданы скриптом `apps/web/scripts/reset-homework-generate-new.mjs`: **54 задания** (18 на класс: 13 выполняемых + 5 внешних сервисов) + **390 сдач** (240 `homework_submissions` + 150 `test_submissions`). Типы `bundle`/`h5p` не используются. В текстовых заданиях НЕ просят прикреплять файл — только «напиши ответ».
   - Ученик видит секцию **«Ваша отправленная работа»** по всем типам: read-only код (`programming`), текст под «Ваш ответ» (`file`), результат «правильных/из скольких» (`test`).
   - **Внешние сервисы (wokwi/geogebra/desmos/codesandbox/learningapps) снова СДАЮТСЯ**: ученик прикрепляет фото (бакет `homework-files`) и/или ссылку (в `answer_text`), видит превью+ссылку+статус; учитель видит то же в `ReviewModal` и ставит оценку через `gradeSubmission`.
   - Разбор ответов теста **«Просмотреть свои ответы» больше не пуст** — скрипт `apps/web/scripts/backfill-test-answers.mjs` **запущен** (150 `started_at` + 750 `test_answers` согласованно с оценками); UI показывает выбор ученика + верный вариант.
   - Починен bundle-баг «0 из N подзадач выполнено» (debounce-сохранение флашится перед отправкой).
8. **Дашборд ученика**: «Мой прогресс» считается от ЗАДАНИЙ (сдано/выдано за всё время), клик → модалка с разбивкой по предметам + средний балл. «Факт дня» не обрезается, заглушка «Новый урок» убрана, AI-кнопка скрыта на `/messages`.
9. **Страница «Оценки»**: фильтр Все/За задания/За урок + модалка деталей (**только просмотр**, правка учителем — техдолг). **Починен критический баг:** `getStudentGrades()` резолвил предмет из `groups.subject` (мусорная заглушка = `'programming'` у всех групп) → все оценки были «Программирование». Переведено на `homework.subject_id`/`lessons.subject_id` → `subjects.name`. Проверено: было 47 оценок все `programming`, стало 5 предметов. Фикс применился и к родительскому виду (функция общая).

## 5. ИЗВЕСТНЫЕ БАГИ / НЕЗАКРЫТЫЕ ХВОСТЫ

Критичных, блокирующих демо, багов **нет** — веб стабилен. Остаточные (не горят, вынесены в техдолг §7):

- **`projects.subject_id` — схемный пробел.** У таблицы `projects` нет FK на предмет, только текстовый `projects.subject` из мусорного `groups.subject`. Сейчас 0 оценённых проектов → не проявляется; при появлении проектных оценок предмет будет неверным. Чинится миграцией (техдолг).
- **H5P: домен и вход.** Проект `snr-eduos-h5p` на Vercel жив, но `h5p.eduos.snruz.uz` не резолвится (DNS/привязка), `/library` отдаёт `307` на логин. H5P исключён из заданий (тип `h5p` не используется).
- **Middleware пишет предупреждение Supabase** на каждый запрос: `getSession()` вместо `getUser()` (данные пользователя из cookie без серверной проверки). RLS всё равно защищает — не горит.

## 7. ТЕХНИЧЕСКИЙ ДОЛГ (не блокирует демо, держать в уме)

- **Ротация `SUPABASE_SERVICE_ROLE_KEY`.** Проект на legacy-ключах (`eyJ...`) — ротация по одному невозможна, только roll JWT-секрета целиком (меняет `anon`+`service_role`, разлогинивает всех, ломает прод до обновления ключей в Vercel). Чистый путь — переезд на новый формат `sb_secret_...` (уже доступен в Dashboard), требует проверки всех серверных роутов.

Упирается в осознанную security-границу (`PreLessonView` намеренно не видит `lesson_stages` до старта). Один лишний round-trip уже убран (fire-and-forget) — нужен кэш signed URL либо пересмотр границы.
- **Правка оценок учителем прямо в модалке страницы «Оценки»** — сейчас там только просмотр.
- **Миграция `projects.subject_id`** (см. §5) — следующий свободный номер **147**.
- **`students.curator_id`** («персональный куратор») не покрыт автосозданием direct-чатов — сейчас 0 расхождений с `groups.teacher_id`, эффекта нет, задел на будущее.

Заказчику **НЕ понравился** текущий дизайн из Claude Design — нужен современный стиль с эффектом **Liquid Glass** (стекло / блюр / полупрозрачность) во всех меню. **Текущие экраны мобилки удаляем и делаем заново.**

**ПОРЯДОК СТРОГИЙ, не нарушать:**
1. **Первым промтом** — улучшение дизайна во всех меню **в Claude Design** (не в коде).

3. И **только когда дизайн соответствует** — настройка backend.

- **iOS и Android — ОДИНАКОВЫЙ дизайн**, различия только в отступах. Учесть: Liquid Glass родной для iOS; на Android блюр тяжелее и выглядит иначе — на слабых устройствах возможны тормоза (закладывать fallback/упрощение).
