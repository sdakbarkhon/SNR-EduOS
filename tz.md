\# ТЗ — SNR EduOS: Платформа для ученика (Web/PWA, LMS-уровень)

\#\# 1\. ОБЩАЯ ИНФОРМАЦИЯ

**Название модуля**: SNR EduOS — Student Portal  
**Тип**: Web Application / PWA  
**Целевой пользователь**: ученик частной школы/учебного центра SNR  
**Цель**: дать ученику единую точку доступа к расписанию, посещаемости, успеваемости, домашним заданиям, оплатам и коммуникации с преподавателем — заменяя Excel/WhatsApp/бумажные журналы.  
**Связь с другими модулями**: использует общую базу данных с Admin Panel и Teacher Workspace (Supabase). Роль `student`, доступ ограничен RLS-политиками (видит только свои данные и данные своих групп).

\---

\#\# 2\. АРХИТЕКТУРА И СТЕК

\- **Backend**: Supabase (Postgres, Auth, Storage, Realtime)  
\- **Frontend**: Next.js (App Router) \+ Tailwind, PWA-обёртка (manifest \+ service worker для offline-кэша расписания/материалов)  
\- **Auth**: Supabase Auth (email/phone \+ пароль или magic link), роль `student` привязана к записи в таблице `students`  
\- **Хранилище файлов**: Supabase Storage (для домашних заданий, материалов уроков)  
\- **Дизайн-система**: SNR EduOS Design System (sidebar 220px, light gradient фон, status chips, плотные карточки, шрифт Inter)

\---

\#\# 3\. ФУНКЦИОНАЛЬНЫЕ МОДУЛИ

\#\#\# 3.1 Авторизация и онбординг  
\- Вход по логину/телефону \+ паролю (создаётся админом при регистрации ученика)  
\- Восстановление пароля  
\- Первый вход: приветственный экран с кратким туром по разделам  
\- Привязка к группам/курсам автоматически подтягивается из админки

\#\#\# 3.2 Главная панель (Dashboard)  
KPI-карточки (высота 100-120px, согласно дизайн-системе):  
\- Текущий баланс / статус оплаты (chip: Активен / Должник)  
\- Посещаемость за месяц (%)  
\- Ближайший урок (дата, время, предмет, преподаватель)  
\- Невыполненные домашние задания (счётчик)  
\- Средняя оценка/прогресс по курсу

Блок "Сегодня": список уроков на сегодня с кнопкой "Подключиться" (если онлайн-урок) или статусом "Очно"

\#\#\# 3.3 Расписание  
\- Календарь (неделя/месяц) со всеми уроками ученика  
\- Карточка урока: предмет, время, преподаватель, кабинет/ссылка на онлайн-урок, статус (запланирован/прошёл/отменён)  
\- Клик по уроку → детальная страница урока (материалы, ДЗ, посещаемость)  
\- Фильтр по группе/предмету

\#\#\# 3.4 Посещаемость  
\- История посещаемости по датам: статус (присутствовал/отсутствовал/опоздал)  
\- Группировка по предметам/группам  
\- Общий % посещаемости (общий и по предмету)  
\- Уведомление-плашка, если посещаемость ниже порога (настраивается админом)

\#\#\# 3.5 Домашние задания  
\- Список ДЗ: активные / выполненные / просроченные (status chips)  
\- Карточка ДЗ: описание, материалы (файлы от преподавателя), срок сдачи, предмет  
\- Сдача ДЗ: загрузка файла или текстовый ответ  
\- История проверенных ДЗ: оценка/комментарий преподавателя  
\- Сортировка по дедлайну

\#\#\# 3.6 Оценки и прогресс  
\- Журнал оценок по предметам (таблица: дата, тип работы, оценка, комментарий)  
\- Средний балл по предмету и общий  
\- График динамики успеваемости (по предмету, по времени) — простой line/bar chart  
\- Прогресс по курсу (если курс структурирован по модулям — % завершения)

\#\#\# 3.7 Материалы курса / Учебные ресурсы  
\- Раздел "Материалы" по каждой группе/предмету  
\- Список файлов/ссылок, загруженных преподавателем (PDF, видео, ссылки)  
\- Группировка по темам/урокам  
\- Возможность скачивания (для PWA — кэширование для offline-доступа)

\#\#\# 3.8 Оплаты и баланс  
\- Текущий баланс (read-only, формируется в админке/у бухгалтера)  
\- История платежей: дата, сумма, тип (абонемент/разовая оплата), статус  
\- История списаний (за посещённые занятия, если используется поурочная тарификация)  
\- Статус-чип: Активен / Должник / Заморожен  
\- Без онлайн-оплаты в этой версии — только отображение информации

\#\#\# 3.9 Коммуникация  
\- Простой чат/уведомления от преподавателя или администрации (одностороннее сообщение от школы \+ возможность ответа ученика)  
\- Лента объявлений школы (общие новости, изменения расписания)  
\- Push-уведомления через PWA (Web Push API) для: новое ДЗ, изменение расписания, новая оценка, низкая посещаемость  
\#\#\# 3.10 Профиль ученика  
\- Личные данные (имя, контакты, дата рождения — read-only, редактирование через админа)  
\- Привязанные группы/курсы  
\- Контакты куратора/классного руководителя  
\- Настройки уведомлений (включить/выключить типы push)

\#\# 5\. РОЛИ И ДОСТУП (RLS)

\- Ученик видит только: свои записи в `students`, `attendance`, `homework_submissions`, `grades`, `payments`, `lessons` (через свои группы), `messages` (адресованные ему/его группе)  
\- Ученик НЕ может: редактировать оценки, посещаемость, баланс, изменять расписание  
\- Ученик может: сдавать ДЗ (создавать записи в \`homework\_submissions\`), отвечать в чате, менять настройки уведомлений и пароль

\---

\#\# 6\. СТРУКТУРА БАЗЫ ДАННЫХ (новые/расширенные таблицы)

Дополнительно к таблицам из пилота (\`students\`, `groups`, `attendance`, \`payments\`):

\- `lessons` (group\_id, date, time, topic, materials\_link, status)  
\- `homework` (group\_id, lesson\_id, title, description, due\_date, attachments)  
\- `homework_submissions` (homework\_id, student\_id, submitted\_at, file\_url/text, grade, teacher\_comment, status)  
\- `grades` (student\_id, subject, lesson\_id, score, type, date, comment)  
\- `course_materials` (group\_id, lesson\_id, title, file\_url/link, type)  
\- `messages` (sender\_id, recipient\_id/group\_id, text, created\_at, read\_status)  
\- `announcements` (school\_id, title, text, target\_group, created\_at)  
\- `notification_settings` (student\_id, push\_homework, push\_schedule, push\_grades, push\_attendance)

\#\# 7\. PWA-ТРЕБОВАНИЯ

\- Manifest.json (иконки, название, theme color по бренд-гайду SNR)  
\- Service Worker: кэширование расписания, материалов, статичных ресурсов для offline-просмотра  
\- Web Push для уведомлений  
\- Установка на главный экран (Add to Home Screen) — promt при первом визите  
\- Адаптивность: mobile-first (большинство учеников/родителей будут на телефонах), desktop — secondary

\---

\#\# 8\. NON-FUNCTIONAL ТРЕБОВАНИЯ

\- Загрузка главной страницы \< 2 сек на 4G  
\- Поддержка офлайн-просмотра расписания и материалов (последний синхронизированный кэш)  
\- Языки интерфейса: русский (приоритет), возможность добавить узбекский/английский позже (i18n-структура заложить сразу)  
\- Безопасность: RLS на уровне Supabase, токены сессий, без хранения чувствительных данных в localStorage

\---

\#\# 9\. ПРИОРИТИЗАЦИЯ ДЛЯ РЕАЛИЗАЦИИ

**Must-have**: Dashboard, Расписание, Посещаемость, ДЗ (просмотр+сдача), Оплаты/баланс (read-only), Профиль  
**Should-have**: Оценки и прогресс, Материалы курса, Объявления  
**Nice-to-have**: Чат с преподавателем, push-уведомления  
**V2**: Геймификация (баллы/рейтинг), интеграция с Telegram-ботом вместо/вместе с push, мультиязычность UI

\---

**Решение для быстрого запуска**  
Брать Must-have блок (3.1-3.4, 3.8, 3.10) — это закрывает то, что уже валидируется в пилоте у админа/учителей, плюс даёт ученику видимость своих данных. Остальное (ДЗ, оценки, материалы, чат) добавлять по мере того, как Teacher Workspace начнёт генерировать эти данные.

**Что можно отложить**  
Чат, push-уведомления, геймификация, мультиязычность, графики динамики успеваемости — добавить после того, как Must-have протестирован на реальных учениках.

**Следующий лучший шаг**  
Расширить схему БД пилота на таблицы `lessons`, `homework`, `grades`, `homework_submissions` — без них Teacher Workspace не сможет генерировать данные, которые ученик увидит в портале.
